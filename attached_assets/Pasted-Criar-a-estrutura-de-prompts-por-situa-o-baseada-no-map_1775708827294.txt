══ PASSO 1 — Migration: nova tabela isp_situation_prompts ══

Adicione ao schema Drizzle (server/db/schema.ts ou equivalente):

  export const ispSituationPrompts = pgTable('isp_situation_prompts', {
    id: uuid('id').defaultRandom().primaryKey(),
    workspace_id: uuid('workspace_id').notNull(),
    situation_code: varchar('situation_code', { length: 10 }).notNull(),
    situation_name: varchar('situation_name', { length: 100 }).notNull(),
    agent: varchar('agent', { length: 30 }).notNull(),
    context_template: text('context_template').notNull(),
    escalate_always: boolean('escalate_always').default(false),
    is_active: boolean('is_active').default(true),
    created_at: timestamp('created_at').defaultNow(),
    updated_at: timestamp('updated_at').defaultNow(),
  });

Índice único em (workspace_id, situation_code).
Rode a migration via POST /api/n8n/setup-db.

══ PASSO 3 — Rotas API para situações ══

Em server/routes/isp.ts, adicione:

GET  /api/isp/agent/situations
  → lista todas as situações do workspace (filtradas por workspace_id)

PUT  /api/isp/agent/situations/:situationCode
  → atualiza context_template ou is_active de uma situação

POST /api/isp/agent/situations/seed
  → executa seedSituationPrompts para o workspace logado

Rode o seed para o workspace da ConexãoNet após a migration.
