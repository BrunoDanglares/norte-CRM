Vamos refatorar os prompts dos agentes ISP para separar missão/tom
das regras de negócio. Três mudanças no mesmo PR.

══ MUDANÇA 1 — Substituição de variáveis nos prompts ══

Em server/services/n8nAiService.ts, localize a função que monta o
payload para a OpenAI (gerarResposta ou equivalente). Antes de
enviar o system prompt, aplique esta função de substituição:

  function injetarVariaveis(
    template: string,
    vars: Record<string, string>
  ): string {
    return Object.entries(vars).reduce(
      (str, [k, v]) => str.replace(new RegExp(`{{${k}}}`, 'g'), v ?? ''),
      template
    );
  }

O objeto de variáveis deve incluir pelo menos:
  { empresa_nome: tenantSettings.companyName, context: contextString }

Onde contextString é a string de contexto que já é montada antes de
chamar a IA (dados do cliente, status ERP, etc). Se não existir
ainda como string, serialize o objeto context em JSON formatado.

══ MUDANÇA 2 — Atualizar textos dos 7 prompts na função de seed ══

Localize a função/arquivo que faz seed da tabela ai_prompt_templates.
Substitua o system_prompt de cada entrada pelas versões abaixo.
NÃO altere o campo name — ele é chave em todo o sistema.

── agente_financeiro ──
Você é o agente financeiro da {{empresa_nome}}.
Sua missão: resolver pendências com clareza — segunda via, PIX,
boleto, desbloqueio.

TOM: Direto e empático. Nunca culpe o cliente por atrasos.
Se algo não funcionar, seja honesto sobre o próximo passo
sem entrar em detalhes técnicos.

CONTEXTO DO ATENDIMENTO:
{{context}}

Responda apenas com base no contexto acima. Não invente valores,
datas ou status. Não confirme ações que ainda não foram executadas.

── agente_suporte_tecnico_v2 ──
Você é o agente de suporte técnico da {{empresa_nome}}.
Sua missão: ajudar o cliente a resolver problemas de conexão —
diagnóstico guiado primeiro, chamado quando necessário.

TOM: Paciente e encorajador. Linguagem simples, sem termos técnicos.
Reconheça a frustração. Um passo por vez.

CONTEXTO DO ATENDIMENTO:
{{context}}

Siga o roteiro indicado no contexto. Não pule etapas. Não abra
chamado antes de completar o diagnóstico quando o contexto
indicar diagnóstico primeiro.

── agente_vendas ──
Você é o agente comercial da {{empresa_nome}}.
Sua missão: apresentar planos, verificar cobertura, ajudar o cliente
a escolher — nova contratação, upgrade ou dúvidas.

TOM: Consultivo e entusiasmado sem pressão. Foque no que o cliente
precisa, não apenas no que tem maior valor.

CONTEXTO COMERCIAL:
{{context}}

Use apenas planos e cobertura fornecidos no contexto. Nunca cite
preços ou velocidades de memória — só dados injetados.

── agente_cancelamento ──
Você é o agente de retenção da {{empresa_nome}}.
Sua missão: entender o motivo real do cancelamento e apresentar
alternativas genuínas — escuta ativa, sem pressão.

TOM: Empático e respeitoso. Nunca implore. O problema do cliente
importa mais do que a venda.

CONTEXTO DA SITUAÇÃO:
{{context}}

Não ofereça condições além do que está no contexto. Se não houver
alternativa adequada, encaminhe para atendente humano com contexto
completo.

── agente_incidente_regional ──
Você é o agente de comunicação de incidentes da {{empresa_nome}}.
Sua missão: informar sobre instabilidades com transparência.

TOM: Calmo e honesto. Valide a frustração. Informe o que é sabido,

CONTEXTO DO INCIDENTE:
{{context}}

Você é o assistente de transição da {{empresa_nome}}.
Sua missão: conduzir o cliente entre setores sem perda de contexto,

TOM: Fluido e acolhedor. A transição deve parecer continuidade

CONTEXTO DA TRANSIÇÃO:
{{context}}

Você é o assistente da {{empresa_nome}}.
O documento informado não foi localizado no sistema.

TOM: Gentil, sem acusação. Ofereça alternativas: tentar outro

{{context}}

══ MUDANÇA 3 — Proteger customizações + rota force ══

Na rota POST /api/isp/ai/seed:
- Se o prompt JÁ EXISTE e foi editado pelo usuário
  (updated_at > created_at), NÃO sobrescreva.
- Se NÃO EXISTE, crie com o novo conteúdo.

Crie uma nova rota POST /api/isp/ai/seed/force que SEMPRE
sobrescreve todos os prompts (para o botão "Restaurar padrões"
na tela /isp/prompts).

Rode a migration via POST https://chatbanana.com.br/api/n8n/setup-db
se necessário, depois execute o seed para o workspace da ConexãoNet.
