PROMPT 7 — Ajustes finais e proteções
Fazer os ajustes finais de segurança e UX no sistema de parceiros.

=== 1. PROTEÇÃO DE ROTAS ===

No ProtectedRoute.tsx (ou criar lógica similar), adicionar suporte 
a requiredAccountType opcional:

<ProtectedRoute 
  path="/parceiros/painel" 
  component={ParceiroPainel}
  requiredAccountType="gestor" 
/>

Se usuário logado tiver account_type !== 'gestor' e tentar acessar 
/parceiros/painel, redirecionar para /dashboard com toast 
"Acesso disponível apenas para contas de parceiro".

=== 2. SEGURANÇA NO BACKEND ===

Em todas as rotas /api/partner/*, adicionar validação:
  const workspace = await storage.getWorkspace(req.workspaceId);
  if (workspace.account_type !== 'gestor') {
    return res.status(403).json({ 
      error: 'Acesso restrito a contas de parceiro' 
    });
  }

Na rota POST /api/partner/clients, adicionar validação:
  const stats = await storage.getPartnerStats(req.workspaceId);
  if (stats.totalClients >= workspace.max_sub_workspaces) {
    return res.status(403).json({
      error: 'Limite de clientes atingido. Faça upgrade do plano.',
      code: 'CLIENT_LIMIT_REACHED'
    });
  }

=== 3. SIDEBAR — CONTEXTO DE IMPERSONATION ===

Quando sessionStorage tiver 'partnerToken' (modo impersonation):
- Adicionar no final da sidebar um item fixo com fundo amber:
  "👁 Modo visualização" 
  "[Nome do cliente]"
  Botão "Sair" que executa o mesmo fluxo de voltar do banner

=== 4. LANDING PAGE ===

Na página landing.tsx, atualizar a seção de planos/CTA para 
mencionar o modelo de parceiro.

Adicionar seção "Para Gestores de Automação" com:
- Título: "Revenda ChatBanana para seus clientes"

- CTA "Quero ser parceiro" → /register (que vai mostrar a 
  tela de seleção de perfil)

=== 5. EMAIL DE BOAS-VINDAS (opcional se tiver serviço de email) ===

Se o sistema já tiver envio de email configurado, ao criar 
conta de gestor, enviar email com:
