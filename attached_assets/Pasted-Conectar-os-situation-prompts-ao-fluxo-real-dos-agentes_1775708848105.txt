Conectar os situation prompts ao fluxo real dos agentes e
adicionar edição na tela /isp/prompts.

══ PASSO 1 — Carregar situation prompt quando situação é detectada ══

Em server/services/n8nAiService.ts, atualize a função gerarResposta
para aceitar um parâmetro opcional situationCode?: string.

Quando situationCode for fornecido:
1. Busque o registro em isp_situation_prompts onde
   workspace_id = workspaceId AND situation_code = situationCode
2. Se encontrado e is_active = true:
   - Substitua as variáveis {{placeholder}} do context_template
     com os dados disponíveis no objeto context
   - Prefixe o context existente com:
     "SITUAÇÃO IDENTIFICADA: [situation_name]\n[context_template_resolvido]\n\n"
3. Se escalate_always = true no registro:
   - Adicione ao context: "\nIMPORTANTE: Esta situação requer escalonamento
     para atendente humano após coleta do contexto necessário."

══ PASSO 3 — Frontend: aba Situações na tela /isp/prompts ══

Em client/src/pages/isp-prompts.tsx:

Adicione uma segunda aba "Situações" ao lado da aba atual "Agentes".

Na aba Situações:
- Mostre as situações agrupadas por agente (Financeiro / Suporte / Comercial)
- Cada situação tem: nome, código (ex: F1), toggle is_active, e textarea
  para editar o context_template
- Badge vermelho "Sempre escala" nas situações com escalate_always = true
- Botão "Salvar" individual por situação (PUT /api/isp/agent/situations/:code)
- Botão global "Restaurar padrões" que chama POST /api/isp/agent/situations/seed

O layout segue o mesmo padrão visual da aba de Agentes existente
(sidebar com lista de situações à esquerda, editor à direita).
