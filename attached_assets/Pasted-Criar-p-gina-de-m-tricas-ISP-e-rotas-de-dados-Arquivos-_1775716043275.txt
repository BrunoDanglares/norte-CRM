══ PASSO 1 — Rotas de métricas ══

Em server/routes/isp.ts adicione:

GET /api/isp/metrics/overview
  Query params: days=7 (default), workspaceId (super_admin)
  Retorna:

  {
    period: { start, end, days },
    sessions: {
      total: number,
      resolved: number,
      escalated: number,
      resolution_rate: number
    },
    inputs: {
      text: number,
      button: number,
      audio: number
    },
    guard: {
      total_blocks: number,
      total_redirects: number,
      most_blocked_action: string
    },
    fluid_swaps: number,
    top_situations: [
      { code, name, agent, activations, escalations }
    ] // top 10 por activations
  }

  SQL principal:
    SELECT
      COUNT(*) as total,
      SUM(CASE WHEN resolved THEN 1 ELSE 0 END) as resolved,
      SUM(CASE WHEN escalou_humano THEN 1 ELSE 0 END) as escalated,
      SUM(guard_blocks) as guard_blocks,
      SUM(guard_redirects) as guard_redirects,
      SUM(fluid_swaps) as fluid_swaps
    FROM isp_session_metrics
    WHERE workspace_id = $1
      AND created_at >= NOW() - ($2 || ' days')::interval

GET /api/isp/metrics/situations
  Query params: days=30, agent (filtro opcional)
  Retorna array de isp_situation_stats agrupado por
  situation_code, somando activations, escalations, guard_blocks.
  Join com isp_situation_prompts para trazer situation_name.

GET /api/isp/metrics/timeline
  Query params: days=14
  Retorna uma linha por dia com total de sessões,
  escalações e fluid_swaps. Para o gráfico de linha.
