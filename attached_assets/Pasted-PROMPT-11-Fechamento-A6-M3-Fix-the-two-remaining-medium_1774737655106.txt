## FIX 1: Add timeouts to all external API calls (A6)

Files: server/routes.ts (now split into modules), server/services/automationEngine.ts

PROBLEM: External API calls to Z-API, Meta, OpenAI, SGP, IXC have no timeout.
A slow/unresponsive external service can hang a request indefinitely.

Create a utility function in server/utils/helpers.ts:
```typescript
export function fetchWithTimeout(
  url: string,
  options: RequestInit = {},
  timeoutMs: number = 30000
): Promise<Response> {
  const controller = new AbortController();
  const timer = setTimeout(() => controller.abort(), timeoutMs);
  
  return fetch(url, { ...options, signal: controller.signal })
    .finally(() => clearTimeout(timer));
}
```

Find all raw fetch() calls in server/ that call external services:
- Z-API calls (api.z-api.io)
- OpenAI calls (api.openai.com)
- Any other external fetch() in server/

Replace with fetchWithTimeout:
```typescript
// BEFORE:
const response = await fetch('https://api.z-api.io/...', options);

// AFTER:
const response = await fetchWithTimeout('https://api.z-api.io/...', options, 30000);
```

Note: SGP and IXC adapters already have AbortController timeouts — skip those.
Note: Meta API calls in meta-whatsapp.ts already have metaFetch with timeout — skip those.
Focus only on remaining raw fetch() calls in route modules and automation engine.
