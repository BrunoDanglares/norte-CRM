Substitua o catch vazio por log estruturado:

  } catch (err: any) {

CORREÇÃO 3 — seedDefaultPipelines idempotente
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Localize a função seedDefaultPipelines() no projeto
(provavelmente em server/index.ts ou server/utils/helpers.ts).

Deve ter algo como:
  try {
    await db.insert(pipeline_stages).values([...]);
  } catch {
    // silencioso na colisão
  }

Substitua por verificação explícita antes de inserir:

  async function seedDefaultPipelines(workspaceId: string) {
    // Verifica se já existem stages para esse workspace
    const existing = await db
      .select({ id: pipelineStages.id })
      .from(pipelineStages)
      .where(eq(pipelineStages.workspaceId, workspaceId))
      .limit(1);

    if (existing.length > 0) return; // já seeded, não duplicar

    // Insere os stages padrão
    await db.insert(pipelineStages).values([
      // ... stages existentes no código atual
    ]);
  }

Isso elimina o try/catch silencioso e torna o seed explicitamente
idempotente — roda N vezes sem efeito colateral.
