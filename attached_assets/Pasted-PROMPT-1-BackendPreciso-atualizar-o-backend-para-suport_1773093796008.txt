PASSO 5 — Criar rota POST /api/perfil/alterar-senha

- Protegida com requireAuth
- Body: { senhaAtual, novaSenha, confirmarSenha }
- Validações:
  - Se qualquer campo faltando: status 400,
    { error: "Todos os campos são obrigatórios" }
  - Se novaSenha !== confirmarSenha: status 400,
    { error: "A nova senha e a confirmação não coincidem" }
  - Se novaSenha.length < 6: status 400,
    { error: "A nova senha deve ter pelo menos 6 caracteres" }
  - Busca usuário no banco pelo req.user.id
  - Chama verifyPassword(senhaAtual, user.password)
  - Se senha atual incorreta: status 401,
    { error: "Senha atual incorreta" }
- Gera hash da nova senha com hashPassword(novaSenha)
- Atualiza password no banco
- Resposta: { ok: true, message: "Senha alterada com sucesso" }

PASSO 6 — Criar rota POST /api/perfil/avatar

- Protegida com requireAuth
- Como não há infraestrutura de upload, aceita URL via body:
  Body: { avatarUrl: "https://..." }
- Valida que avatarUrl começa com http/https
- Atualiza campos avatar e avatarUrl no banco para req.user.id
- Resposta: { ok: true, data: { avatarUrl } }

PASSO 7 — Criar rota DELETE /api/perfil/conta

- Protegida com requireAuth
- Body: { senha } — confirma senha antes de excluir

- Se senha incorreta: status 401, { error: "Senha incorreta" }
- Não permite excluir se role === 'admin' e for o único admin:

- Marca usuário como INACTIVE em vez de excluir fisicamente
  (preserva histórico de leads e conversas)
- Resposta: { ok: true, message: "Conta desativada com sucesso" }
