## SGP API — Real Documentation

### Authentication
- Method: Token/App via form-data OR Basic Auth
- Endpoint uses: token + app (application name) sent in request body as form-data
- OR Basic Auth with username/password in Authorization header (Base64)
- Recommended for ChatBanana: send token + app as query params or form-data

### Endpoint: Consultar Cliente (URA)
POST {baseUrl}/api/ura/consultacliente/

Form-data body:
- token: [your token]
- app: [your app name]  
- cpf_cnpj: [cpf or cnpj] (at least one filter required)

### Endpoint: Consultar Títulos (URA)
POST {baseUrl}/api/ura/consultartitulos/

Form-data body:
- token: [your token]
- app: [your app name]
- contrato: [contratoId from consultacliente response]

Response structure:

  "msg": "Título(s) Localizado(s)",

### Endpoint: Gerar PIX / 2ª Via (URA)
POST {baseUrl}/api/ura/tituloPix/

Form-data body:
- token: [your token]
- app: [your app name]
- seis: [contratoId] -- ID do contrato para gerar/retornar PIX do título

Response:
{
  "status": 1,
  "razaoSocial": "NOME",
  "protocolo": "200429143349",

### Endpoint: Enviar Fatura (URA)
POST {baseUrl}/api/ura/enviafatura/

Form-data body:
- token: [your token]
- app: [your app name]
- seis: [contratoId]
- mensagem: [optional custom message]

### Endpoint: Abrir OS (Suporte)
POST {baseUrl}/api/suporte/abriros/ (check exact path in SGP docs)

Form-data body:
- token: [your token]
- app: [your app name]
- contrato: [contratoId]
- assunto: [subject]
- descricao: [description]

Response:

### 1. Fix authentication
Change from Bearer token header to form-data with token + app:
```typescript

// Update ISPCredentials in isp-provider.interface.ts to add:
// sgpApp?: string  -- the application name registered in SGP

// In sgpFetch, send as form-data:

formData.append('app', credentials.sgpApp || 'ChatBanana');
// + any additional params

  method: 'POST',
  body: formData
  // NO Authorization header needed for Token/App method

```

### 2. Fix searchCustomerByCPF
- Change to POST /api/ura/consultacliente/
- Send CPF as form-data field 'cpf_cnpj'
- Map response: response.contratos[0]
- Map fields:
  - id: String(contrato.clienteId)
  - nome: contrato.razaoSocial

### 5. Important: contratoId vs clienteId
The SGP uses TWO different IDs:
- clienteId: identifies the person/customer
- contratoId: identifies the contract/service (needed for invoice queries)

Update ISPCustomer interface to include:
- contratoId: string (stored alongside id=clienteId)

OR store contratoId in a separate field so getOpenInvoices can use it correctly.

The simplest approach: make customerId in getOpenInvoices accept EITHER clienteId or contratoId,
and store contratoId in the customer object returned by searchCustomerByCPF.
