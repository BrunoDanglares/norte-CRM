1. SEGURANÇA & MULTI-TENANCY
🔴 CRÍTICO — S01: Webhook Meta sem validação de assinatura X-Hub-Signature-256
Arquivo: server/routes/webhook-meta.ts, linhas 30-44
Descrição: O endpoint POST /api/webhook/meta aceita qualquer payload sem validar o header X-Hub-Signature-256. Qualquer atacante pode enviar payloads forjados, criar conversas falsas, ou injetar mensagens no sistema.
Correção recomendada:
import crypto from "crypto";
function verifyMetaSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.headers["x-hub-signature-256"] as string;
  if (!signature) return res.sendStatus(403);
  
  const expectedSig = "sha256=" + crypto
    .createHmac("sha256", process.env.WHATSAPP_APP_SECRET!)
    .update(JSON.stringify(req.body))
    .digest("hex");
  
  if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expectedSig))) {
    return res.sendStatus(403);
  }
  next();
}
router.post("/", verifyMetaSignature, (req, res) => { ... });

3. BACKEND & ROTAS
🟡 ALTO — B01: Cron jobs sem proteção contra execução simultânea
Arquivo: server/services/isp-schedulers.ts, linhas 33-39
Descrição: Os crons de billing (08:00) e birthday (09:00) não têm lock. Se o processo reiniciar no meio da execução ou houver múltiplas instâncias, o mesmo job pode rodar em paralelo, enviando mensagens duplicadas.
Correção: Implementar lock via tabela no banco:
async function withLock(lockName: string, fn: () => Promise<void>) {
  const acquired = await db.execute(
    sql`SELECT pg_try_advisory_lock(hashtext(${lockName}))`
  );
  if (!acquired.rows[0].pg_try_advisory_lock) return;
  try { await fn(); } finally {
    await db.execute(sql`SELECT pg_advisory_unlock(hashtext(${lockName}))`);
  }
}

🟢 MÉDIO — W03: Sem retry automático para erros transitórios da Meta
Arquivo: server/services/meta-whatsapp.ts, linhas 62-72
Descrição: metaFetch tem timeout de 10s mas não implementa retry para erros 429 (rate limit) ou 500 (erro temporário da Meta).
Correção: Implementar retry com backoff exponencial usando p-retry:
import pRetry from 'p-retry';
async function metaFetchWithRetry(url, options) {
  return pRetry(() => metaFetch(url, options), {
    retries: 3,
    minTimeout: 1000,
    factor: 2,
    onFailedAttempt: (err) => console.warn(`Meta retry ${err.attemptNumber}/${err.retriesLeft}`),
  });
}
