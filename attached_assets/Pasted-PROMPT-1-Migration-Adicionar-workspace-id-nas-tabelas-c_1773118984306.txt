PROMPT 1 — Migration: Adicionar workspace_id nas tabelas coreImplemente o isolamento real de dados multi-tenant no FlowCRM.
Hoje as tabelas core não têm workspace_id e todos os dados são compartilhados.

## CONTEXTO
- A tabela workspaces já existe no banco
- Algumas tabelas já têm workspace_id (users, conexoes, etc.)
- O usuário autenticado tem tenantId disponível via JWT (req.user.tenantId)
- Existe um workspace padrão com id = 1

---

## PARTE 1: Migration SQL

Execute as seguintes alterações no banco:

### Adicionar workspace_id nas tabelas core

ALTER TABLE leads ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE conversations ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE messages ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE deals ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE appointments ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE transactions ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE automations ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE campaigns ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
ALTER TABLE pipeline_stages ADD COLUMN IF NOT EXISTS workspace_id INTEGER REFERENCES workspaces(id);
