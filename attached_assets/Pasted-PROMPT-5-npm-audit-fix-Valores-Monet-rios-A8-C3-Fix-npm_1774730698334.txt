## FIX 2: Monetary values precision (C3)

File: shared/schema.ts

PROBLEM: valor fields use real() (float) causing precision errors.

FIND these fields:
- leads.valor (or similar name)
- deals.valor
- transactions.valor
- Any other monetary fields using real() or doublePrecision()

SOLUTION: Change to numeric with precision:
```typescript
// BEFORE:
valor: real("valor"),

// AFTER:
valor: numeric("valor", { precision: 10, scale: 2 }),
```

Find ALL monetary fields in the schema and update them.
Look for fields named: valor, preco, desconto, mensalidade, total, subtotal,
invoice_amount, valor_pago, taxa.

After updating schema:
1. Run npm run db:push to apply changes
2. Verify existing data was not corrupted (numeric and real are compatible in PostgreSQL)
3. Update any TypeScript code that treats these as number type:

   - Add parseFloat() where needed: parseFloat(lead.valor || '0')
   - Or use a helper: const toMoney = (v: string | null) => parseFloat(v || '0')

IMPORTANT: Do NOT change isp_billing_logs.invoice_amount or 
isp_automation_logs fields — those are already numeric(10,2).
Only fix the core tables: leads, deals, transactions.
