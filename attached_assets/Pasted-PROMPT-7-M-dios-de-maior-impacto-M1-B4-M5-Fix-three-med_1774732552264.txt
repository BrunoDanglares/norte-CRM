## FIX 2: Add missing unique constraints (B4)

File: shared/schema.ts

PROBLEM: No unique constraints on contacts(workspace_id, phone) or
messages(workspace_id, wamid) — duplicates can silently accumulate.

Add unique indexes:
```typescript
// After contacts table definition:
export const contactsWorkspacePhoneUnique = uniqueIndex("idx_contacts_workspace_phone_unique")
  .on(contacts.workspaceId, contacts.telefone);

// After messages table definition:
// Only add this if messages table has a 'wamid' column
// Check schema first — if column exists, add:
export const messagesWamidUnique = uniqueIndex("idx_messages_wamid_unique")  
  .on(messages.workspaceId, messages.wamid);
```

IMPORTANT before running migration:
1. Check for existing duplicate contacts (same workspace_id + phone):
```sql
SELECT workspace_id, telefone, COUNT(*) 
FROM contacts 
GROUP BY workspace_id, telefone 
HAVING COUNT(*) > 1;
```
2. If duplicates exist, deduplicate first (keep the most recent):
```sql
DELETE FROM contacts
WHERE id NOT IN (
  SELECT MAX(id) 
  FROM contacts 
  GROUP BY workspace_id, telefone
);
```
3. Then run npm run db:push to add the unique constraint.

Same process for messages.wamid if duplicates exist.

## FIX 3: Add ON DELETE CASCADE to core tables (M5)

File: shared/schema.ts

PROBLEM: Core tables (leads, contacts, conversations, messages) don't have
CASCADE on their workspace foreign keys. Deleting a workspace would fail
if it has any data.

Find the foreign key references to workspaces in these tables:
```typescript
// BEFORE (no cascade):
workspaceId: uuid("workspace_id").references(() => workspaces.id),

// AFTER (with cascade):
workspaceId: uuid("workspace_id")
  .references(() => workspaces.id, { onDelete: 'cascade' }),
```

Apply to ALL core tables:
- leads
- contacts  
- conversations
- messages
- deals
- appointments
- transactions

Note: ISP tables already have cascade — only fix the core tables.

IMPORTANT: This migration is safe on existing data. CASCADE only affects
future DELETE operations on the workspaces table.

Run npm run db:push after changes.
Confirm server starts and basic CRUD operations still work.
