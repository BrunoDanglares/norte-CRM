
PROMPT 1 — Real-time no Inbox via WebSocket
Implemente mensagens em tempo real no Inbox do FlowCRM usando WebSocket.
Hoje o inbox só atualiza ao recarregar a página.

---

## PARTE 1: Backend — servidor WebSocket

No arquivo principal do servidor (server/index.ts), adicione WebSocket
usando o pacote 'ws' (já disponível no Node.js ou instale: npm install ws @types/ws):
```typescript
import { WebSocketServer, WebSocket } from 'ws'
import { IncomingMessage } from 'http'
import jwt from 'jsonwebtoken'

// Mapa de conexões ativas: workspaceId → Set de WebSocket clients
const wsClients = new Map<string, Set<WebSocket>>()

// Inicializa WSS no mesmo servidor HTTP do Express
const wss = new WebSocketServer({ server: httpServer, path: '/ws' })

wss.on('connection', (ws: WebSocket, req: IncomingMessage) => {
  // Autenticar via token na query string: /ws?token=<jwt>
  const url = new URL(req.url!, `http://${req.headers.host}`)
  const token = url.searchParams.get('token')

  if (!token) { ws.close(1008, 'Token required'); return }

  let workspaceId: string
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!) as any
    workspaceId = decoded.workspaceId || decoded.tenantId
  } catch {
    ws.close(1008, 'Invalid token'); return
  }

  // Registrar cliente no workspace
  if (!wsClients.has(workspaceId)) wsClients.set(workspaceId, new Set())
  wsClients.get(workspaceId)!.add(ws)

  // Ping/pong para manter conexão viva
  const pingInterval = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) ws.ping()
  }, 30000)

  ws.on('close', () => {
    clearInterval(pingInterval)
    wsClients.get(workspaceId)?.delete(ws)
  })

  ws.on('error', () => {
    clearInterval(pingInterval)
    wsClients.get(workspaceId)?.delete(ws)
  })
})

// Função para broadcast — use em qualquer parte do servidor
export function broadcastToWorkspace(workspaceId: string, event: string, data: object) {
  const clients = wsClients.get(workspaceId)
  if (!clients?.size) return

  const payload = JSON.stringify({ event, data, timestamp: new Date().toISOString() })
  clients.forEach(client => {
    if (client.readyState === WebSocket.OPEN) client.send(payload)
  })
}
```

## PARTE 3: Frontend — hook useWebSocket

Crie client/src/hooks/useWebSocket.ts:
```typescript
import { useEffect, useRef, useCallback } from 'react'
import { useAuth } from '../contexts/AuthContext'

type WSEventHandler = (data: any) => void

export function useWebSocket(handlers: Record<string, WSEventHandler>) {
  const { token } = useAuth()
  const wsRef = useRef<WebSocket | null>(null)
  const handlersRef = useRef(handlers)
  handlersRef.current = handlers

  const connect = useCallback(() => {
    if (!token) return
    if (wsRef.current?.readyState === WebSocket.OPEN) return

    const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:'
    const ws = new WebSocket(`${protocol}//${window.location.host}/ws?token=${token}`)

    ws.onopen = () => console.log('[WS] Conectado')

    ws.onmessage = (event) => {
      try {
        const { event: eventName, data } = JSON.parse(event.data)
        handlersRef.current[eventName]?.(data)
      } catch {}
    }

    ws.onclose = (e) => {
      if (e.code !== 1008) {
        // Reconectar após 3 segundos (exceto se token inválido)
        setTimeout(connect, 3000)
      }
    }

    wsRef.current = ws
  }, [token])

  useEffect(() => {
    connect()
    return () => wsRef.current?.close()
  }, [connect])
}
```

## PARTE 4: Usar o hook no Inbox

No componente da página Inbox (client/src/pages/inbox.tsx):
```typescript
import { useWebSocket } from '../hooks/useWebSocket'

// Dentro do componente:
useWebSocket({
  // Nova mensagem recebida ou enviada
  new_message: (data) => {
    const { conversationId, message } = data

    // 1. Se a conversa está aberta, adiciona a mensagem ao histórico
    if (selectedConversation?.id === conversationId) {
      setMessages(prev => {
        // Evitar duplicata
        if (prev.find(m => m.id === message.id)) return prev
        return [...prev, message]
      })
      // Scroll automático para o final
      setTimeout(() => {
        messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })
      }, 50)
    }

    // 2. Atualiza a lista de conversas (última mensagem + unread)
    setConversations(prev => prev.map(conv =>
      conv.id === conversationId
        ? {
            ...conv,
            last_message: message.content,
            last_message_at: message.created_at,
            unread_count: selectedConversation?.id === conversationId
              ? 0  // já está aberta, não incrementa
              : (conv.unread_count || 0) + 1
          }
        : conv
    ).sort((a, b) =>
      new Date(b.last_message_at).getTime() - new Date(a.last_message_at).getTime()
    ))
  },

  // Conversa criada ou atualizada
  conversation_updated: (data) => {
    setConversations(prev => {
      const exists = prev.find(c => c.id === data.conversationId)
      if (!exists) {
        // Nova conversa — recarrega a lista
        fetchConversations()
        return prev
      }
      return prev.map(c =>
        c.id === data.conversationId ? { ...c, ...data } : c
      )
    })
  }
})

// Indicador de conexão (opcional mas recomendado)
// Adicione um ponto verde/vermelho no header do Inbox
// Verde = WebSocket conectado, Vermelho = reconectando
```

PROMPT 2 — Envio real via Z-API no Inbox manual
Corrija o envio manual de mensagens no Inbox para usar a Z-API diretamente.
Hoje o envio salva no banco mas não chama a Z-API.

---

## PROBLEMA ATUAL
A rota POST /api/conversations/:id/messages salva no banco e dispara webhook,
mas não envia a mensagem de verdade pelo WhatsApp via Z-API.
A rota de envio real existe em /api/conexoes/:id/send-text mas não está conectada.

---

## SOLUÇÃO

Na rota POST /api/conversations/:id/messages, após salvar no banco,
adicione o envio real via Z-API:
```typescript
router.post('/api/conversations/:id/messages', requireAuth, async (req: AuthRequest, res) => {
  const workspaceId = resolveWorkspaceId(req)
  const { content, type = 'text' } = req.body

  // 1. Verificar que a conversa pertence ao workspace
  const conversation = await storage.getConversation(req.params.id, workspaceId)
  if (!conversation) return res.status(404).json({ error: 'Conversa não encontrada' })

  // 2. Salvar mensagem no banco
  const message = await storage.createMessage({
    conversation_id: conversation.id,
    content,
    type,
    sender: 'agent',
    workspace_id: workspaceId,
    created_at: new Date()
  })

  // 3. Buscar conexão Z-API ativa do workspace
  const conexao = await db.query(
    `SELECT * FROM conexoes 
     WHERE workspace_id = $1 AND status = 'connected' 
     ORDER BY created_at DESC LIMIT 1`,
    [workspaceId]
  )

  // 4. Enviar via Z-API se conexão disponível
  if (conexao.rows[0]) {
    const { zapi_instance_id, zapi_token } = conexao.rows[0]
    const phone = conversation.contact_phone || conversation.phone

    if (phone) {
      try {
        const zapiRes = await fetch(
          `https://api.z-api.io/instances/${zapi_instance_id}/token/${zapi_token}/send-text`,
          {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              phone: phone.replace(/\D/g, ''),  // remove não-dígitos
              message: content
            })
          }
        )

        if (!zapiRes.ok) {
          const err = await zapiRes.json()
          console.error('[Z-API] Erro ao enviar mensagem:', err)
          // Não retorna erro ao cliente — mensagem já foi salva no banco
        }
      } catch (e) {
        console.error('[Z-API] Falha na requisição:', e)
      }
    }
  }

  // 5. Broadcast WebSocket (para real-time)
  broadcastToWorkspace(workspaceId, 'new_message', {
    conversationId: conversation.id,
    message: { ...message, sender: 'agent' }
  })

  // 6. Atualizar last_message e updated_at da conversa
  await db.query(
    `UPDATE conversations 
     SET last_message = $1, last_message_at = NOW(), updated_at = NOW()
     WHERE id = $2 AND workspace_id = $3`,
    [content, conversation.id, workspaceId]
  )

  return res.status(201).json(message)
})
```
