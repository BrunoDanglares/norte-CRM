## PARTE 2: Padrão de correção

Use este padrão em todos os métodos do storage.ts:

### Listagem
```typescript
// ANTES
SELECT * FROM leads ORDER BY created_at DESC

// DEPOIS  
SELECT * FROM leads WHERE workspace_id = $1 ORDER BY created_at DESC
// params: [workspaceId]
```

### Busca por ID (previne acesso cross-tenant)
```typescript
// ANTES
SELECT * FROM leads WHERE id = $1

// DEPOIS
SELECT * FROM leads WHERE id = $1 AND workspace_id = $2
// Se não encontrar → retorna null (rota retorna 404)
// params: [id, workspaceId]
```

### Criação
```typescript
// ANTES
INSERT INTO leads (name, phone) VALUES ($1, $2)

// DEPOIS
INSERT INTO leads (name, phone, workspace_id) VALUES ($1, $2, $3)
// params: [...campos, workspaceId]
```

### Update / Delete
```typescript
// ANTES
UPDATE leads SET name = $1 WHERE id = $2

// DEPOIS
UPDATE leads SET name = $1 WHERE id = $2 AND workspace_id = $3
// Se 0 rows affected → retorna null (rota retorna 404)
```

## PARTE 3: Corrigir rotas para passar workspaceId

Em server/routes.ts, todas as rotas protegidas devem extrair e passar workspaceId:
```typescript
// Em cada rota autenticada
const workspaceId = resolveWorkspaceId(req)

// Passar para o storage
const leads = await storage.getLeads(workspaceId)
const lead = await storage.createLead({ ...req.body, workspaceId })
const lead = await storage.updateLead(id, req.body, workspaceId)
const lead = await storage.getLead(id, workspaceId)
```

Aplique para todas as entidades:
leads, contacts, conversations, messages, deals,
appointments, transactions, automacoes, campanhas,
pipeline_stages, tags

## PARTE 4: Webhook Z-API — workspace por conexão

No handler POST /api/webhooks/zapi:
```typescript
// Identifique o workspace pela instância Z-API que recebeu a mensagem
const instanceId = webhookPayload.instanceId || webhookPayload.phone

const conexao = await db.query(
  `SELECT workspace_id FROM conexoes 
   WHERE instance_id = $1 OR zapi_instance_id = $1 
   LIMIT 1`,
  [instanceId]
)

const workspaceId = conexao.rows[0]?.workspace_id 
  || process.env.DEFAULT_WORKSPACE_ID  // UUID do workspace padrão

// Use em todas as operações subsequentes do webhook
await storage.createLead({ ...leadData, workspaceId })
await storage.createConversation({ ...convData, workspaceId })
await storage.createMessage({ ...msgData, workspaceId })
```
