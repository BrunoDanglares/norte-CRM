### 1. SECURITY AUDIT

#### 1.1 Authentication & Authorization
- [ ] Are ALL /api/* routes protected with requireAuth middleware?
- [ ] Are there any routes that should require auth but don't?
- [ ] Is the JWT secret strong and stored only in env vars?
- [ ] Are there any hardcoded credentials, tokens, or API keys anywhere in the code?
- [ ] Is the Meta webhook at /api/webhook/meta and Z-API webhook correctly public (no auth)?
- [ ] Are any other routes accidentally public that shouldn't be?

#### 1.2 Multi-tenancy Isolation
- [ ] Do ALL database queries include workspace_id as a filter?
- [ ] Are there any queries that could return data from other workspaces?
- [ ] Does resolveWorkspaceId(req) work correctly in all routes?
- [ ] Are there any direct ID lookups (findFirst by id only) without workspace_id check?
- [ ] Can a user access another workspace's data by manipulating IDs in the URL?

Search for patterns like:
- db.query.*.findFirst({ where: eq(*.id, id) }) WITHOUT workspace_id
- db.select().from(*).where(eq(*.id, ...)) WITHOUT workspace_id
- Any route param :id used in DB query without workspace scoping

#### 1.3 Input Validation
- [ ] Are request bodies validated before being used in DB queries?
- [ ] Are there any SQL injection risks (raw queries without parameterization)?
- [ ] Are file uploads validated for type and size?
- [ ] Are phone numbers sanitized before being used in WhatsApp API calls?
- [ ] Are CPF/CNPJ inputs sanitized before ERP API calls?

#### 1.4 Sensitive Data Exposure
- [ ] Are API tokens (SGP, IXC, Z-API, Meta, OpenAI) ever returned in API responses?
- [ ] Are JWT secrets or passwords ever logged to console?
- [ ] Are there any console.log statements that print sensitive data in production?
- [ ] Is the Meta access_token masked when returning isp_configs or whatsapp_official_connections?

#### 1.5 Webhook Security
- [ ] Is the Meta webhook verify token checked correctly?
- [ ] Is the Z-API webhook protected against spoofing?
- [ ] Are webhook payloads validated before processing?
