  • React 18 + TypeScript + Vite

  • shadcn/ui (70+ componentes)
  • TanStack React Query v5 (staleTime: 2min)

  • Lazy loading com React.lazy + Suspense

🌐 Landing Page

Pagina publica acessivel em /landing. Usuarios nao autenticados sao redirecionados automaticamente para ela. Contém formulario de login integrado e todas as secoes de marketing.

1. Hero Section

Titulo principal com gradiente violet, subtitulo descritivo, dois CTAs ("Comecar gratis" e "Ver demo"), e indicadores de confianca ("+500 empresas", "Suporte 24/7", "99.9% uptime").

2. Formulario de Login/Cadastro

Formulario integrado na landing com campos email e senha. Login via JWT com redirecionamento automatico para o dashboard apos autenticacao. Link para registro de nova conta.

3. Secao de Features (8 funcionalidades)

Cards com icones mostrando: Pipeline Visual (Kanban drag-and-drop), Inbox Omnichannel (WhatsApp + Instagram), Automacoes IA (fluxos visuais com GPT), Relatorios (dashboards em tempo real), Gestao de Equipe (permissoes granulares), Seguranca (criptografia ponta-a-ponta), Chatbot IA (respostas automaticas), Integracao APIs (webhooks + N8n + Zapier).

4. Secao "Como Funciona" (3 passos)

Passos numerados: 1) Conecte seus canais (WhatsApp em 2 minutos), 2) Configure automacoes (fluxos com IA), 3) Gerencie resultados (dashboards e relatorios).

5. Tabela de Precos (3 planos)

Starter: R$197/mes (R$131/ano) - 2 usuarios, 1.000 contatos, WhatsApp basico, 5 automacoes.
Pro: R$497/mes (R$331/ano) - 10 usuarios, contatos ilimitados, omnichannel, IA, campanhas. Badge "POPULAR".
Enterprise: Sob consulta - usuarios ilimitados, API REST, SLA 1h, white-label, onboarding dedicado.

6. Secao de Depoimentos

Cards com avatares, nomes, cargos e depoimentos de clientes satisfeitos com estrelas de avaliacao.

7. FAQ (Perguntas Frequentes)

Accordion expansivel com perguntas e respostas sobre o sistema, integracao, suporte, etc.

8. Footer

Links do sistema, contato, redes sociais e copyright. Design consistente com tema violet.

⚡ Motor de Automacao (25+ Nos)

Motor de execucao real em server/services/automationEngine.ts (3.319 linhas). Processa fluxos JSON com resolucao de variaveis {{lead.nome}}, {{var.x}}, pausas, retomadas, fuzzy matching e integracao com IA.

Basicos (9 nos)
trigger - Gatilho inicial
send_message - Enviar mensagem
send_image - Enviar imagem/PDF
delay - Aguardar (segundos)
condition - Condicao if/else
tag_lead - Marcar lead com tag
assign_agent - Atribuir atendente
update_lead - Atualizar pipeline/estagio
lista_opcoes - Menu de opcoes
end - Fim do fluxo
Avancados (8 nos)
ai_response - Resposta IA (GPT/Gemini)
advanced_condition - Condicao AND/OR
split_ia - Classificacao IA por intencao
set_variable - Definir variavel
wait_event - Esperar evento externo
loop - Repetir com condicao
alerta_interno - Alerta para usuarios
gerar_documento - Gerar doc HTML
Integracoes (6 nos)
webhook - Chamar URL externa
zapier_trigger - Disparar Zapier/Make
n8n_trigger - Disparar N8n
stripe_payment - Cobrar via Stripe
google_calendar - Agendar no GCal
pagamento - Mercado Pago PIX

Funcionalidades do Motor

  • Resolucao de variaveis: {{lead.nome}}, {{lead.telefone}}, {{var.minha_var}}, {{ctx.last_ai_response}}
  • Escopos de variavel: session (fluxo atual), lead (persistente no lead), global (workspace inteiro)
  • Pausa e retomada: lista_opcoes (fuzzy matching), wait_event (client_reply, payment, webhook, timeout)
  • Loop com max_attempts e condicao de parada
  • Split IA com categorias dinamicas e roteamento por branch
  • Alerta interno com fan-out para todos os usuarios
  • Limpeza final de {{placeholders}} nao resolvidos
  • Protecao contra loops (debounce, fromMe/fromApi filter)
  • AI Response com multimidia (imagem, audio, PDF via GPT-4o Vision + Whisper)
  • AI File Attachments e AI Webhooks
  • Upload de arquivos para send_image (imagens + PDF servidos estaticamente)

🔒 Autenticacao & Multi-Tenant

Autenticacao JWT

  • Login via POST /api/auth/login (email + senha)
  • Senha hasheada com crypto.scryptSync
  • JWT com jsonwebtoken (expira em 7 dias)
  • Token armazenado em localStorage
  • Bearer token em todos os requests
  • 3 middlewares: requireAuth, requireAuthOrToken, requireApiToken
  • ProtectedRoute component no frontend

Multi-Tenant (Workspaces)

  • Tabela workspaces com UUID
  • Cada usuario pertence a um workspace
  • Todas as queries filtram por workspaceId
  • Isolamento completo de dados entre tenants
  • Convite de usuarios com token temporario
  • Permissoes granulares: admin, manager, agent
  • Registro cria workspace + usuario admin + seed
📡 WebSocket & Tempo Real

WebSocket Server (/ws)

  • Servidor ws nativo na rota /ws
  • Autenticacao JWT no handshake
  • Broadcasting por workspace (isolamento multi-tenant)
  • Eventos: new_message, conversation_updated, lead_updated
  • Auto-connect/reconnect no cliente
  • Hook useWebSocket no frontend
  • Notificacoes em tempo real via polling (15s) + WebSocket
